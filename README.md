# INCOTO (투자관제탑) 프로젝트 분석 정리

> 분석 대상: 2015~2017년 재무 투자성과 평가 시스템 (1차 ~ 3차 개발)

| **프로젝트명** | INCOTO (Investment Control Tower) — 투자관제탑      |
| **목적**    | 재무설계사가 고객의 자산·투자 포트폴리오를 통합 관리하고 성과를 분석하는 웹 시스템 |

## 3. 아키텍처 구조

```
[Browser] → JSP
    ↕ Ajax / Form Submit
[Controller] (Spring MVC)
    ↕ Service Layer
[Service / ServiceImpl]
    ↕ DAO Layer
[DataAccessObject] + MyBatis XML Mapper
    ↕
[MySQL DB]
```

### 패키지 구조 (net.su.*)

계층형 아키텍처(Controller → Service → DAO → VO) 기반으로 아래 도메인 영역으로 구성:

| 패키지 | 담당 도메인 |
|--------|------------|
| `login` | 로그인, 회원관리, 게시판, 암호화 |
| `customer` | 고객, 가족구성원, 목표, 설계자 관리 |
| `asset` | 자산 전체 CRUD (유동성·현금성·채권·주식·부동산·대출·보험·카드 8종) |
| `cusPage` | 고객 전용 포트폴리오 조회 페이지 |
| `output` | 투자 성과 분석 및 출력 화면 |

## 4. 주요 기능

### 자산 관리
- 8가지 자산 유형(유동성·현금성·채권형·주식형·부동산·대출·보험·신용카드) CRUD
- 자산별 계약·납입·만기·수익자 정보 통합 관리
- 벤치마크 지수 등록 및 관리 (국내외 주요 지수)

### 투자 성과 분석
- **FCR (미래현금소요)**: 고객 인생 목표별 필요 자금과 현재 적립 상황 비교 분석
- **IRR (내부수익률)**: 투자자산별 현금흐름 기반 실제 수익률 계산 및 벤치마크 비교
- **투자매트릭스**: 자산유형 × 투자등급 기반 포트폴리오 구성 시각화
- **TAA (전술적 자산배분)**: 현재 자산배분 vs 목표 배분 비교 및 리밸런싱 제안

### 지출·현금흐름 분석
- 지출 카테고리별 집계 및 월별 현금흐름 분석
- 연간 지출·정기비용 관리
- 카드지급 연동 추적

### 기타
- 고객 전용 포트폴리오 조회 페이지 (별도 접근 권한)
- 집계 데이터 엑셀 출력 지원
- 시계열 재무지표 추세 분석

## 5. 개발 이력별 주요 변경사항

### 1차 개발 (2015)
- 핵심 자산관리 DB 설계 및 기본 CRUD 구현
- 계약서 기준 기능명세서 작성 및 ERD 설계
- Spring MVC + MyBatis 기반 프로젝트 구조 수립

### 2차 개발 (2016)
- 분석·출력 기능 대거 추가 (FCR, TAA, 투자매트릭스, 투자결과, 현금흐름 등)
- 전체 Java 소스 약 200여 개 파일로 확장
- Bootstrap 기반 반응형 UI 완성

### 3차 추가개발 및 유지보수 (2017)
- 재무전문가 소개 홈페이지 추가
- 고객 계좌·메모 관리 기능 신규 추가
- PMT 계산 기능 (목표금액 ↔ 월납입금 자동 계산)
- 상품별 투자 Tracking 기능 추가
- 시점별 데이터 스냅샷 저장 기능 추가
